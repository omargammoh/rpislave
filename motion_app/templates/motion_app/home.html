Documentation

This app runs another software called motion

Definitions

  • Motion

    The app processes the images of the camera, when there is a change in the images of more than {{threshold}} pixels, for {{minimum_motion_frames}} frames in a row, then a motion is detected.

  • Event

    An event is a series of motions that occure within a small timeframe. The event ends when there is no motion for {{gap}} seconds

Services

  • motion-triggered video files

    The app records the motions from {{pre_capture}} frames before the motion, till {{post_capture}} frames after the motion.

    Motions that are close to each other (i.e. within the same event) are saved in one file

    The {{ffmpeg_video_codec}} codec is used to save the file Files are saved in the with name {{movie_filename}} in the {{target_dir}} directoy

  • motion-triggered images

    If {{output_normal}} is on, the images where motion is detected are saved. If it's off, they are not saved, If it's first, then the first image of an event is saved, If it's best, then the picture of the most motion in an event is saved. If it's center, then the image with the motion close to the center is saved

    If {{output_motion}} is on, then only the pixels with the moving objects are saved (ghost images)

  • livestream

    Live stream of the camera is also provided on port {{webcam_port}}, access to this could be restricted to localhost if {{webcam_localhost}} is on. If {{ webcam_motion }} is on, then the output stream will be at 1 fps when there is no motion and when there is motion, it goes back up to {{ webcam_maxrate }}

  • control configuration

    on port {{control_port}} the configuration of the motion software can be accessed, this is also restricted to localhost if {{control_localhost}} is on

  • timelapse

    if {{ffmpeg_timelapse}}is not 0, timelapse will be on and the period between the shots is {{ffmpeg_timelapse}} seconds

  • snapshots

    if {{snapshot_interval}}is not 0, snapshots will be on and the period between the shots is {{snapshot_interval}} seconds. The file ill be named {{ snapshot_filename }}
